# Politica de Seguranca

## Reportando Vulnerabilidades

A equipe do VFNews leva a seguranca do projeto a serio. Agradecemos seus
esforcos para divulgar responsavelmente as vulnerabilidades e trabalhamos para
reconhecer e apreciar suas contribuicoes.

Para reportar uma vulnerabilidade de seguranca, por favor, envie um e-mail para
o endereco de contato do mantenedor do projeto. Inclua o maximo de detalhes
possivel:

- Descricao do problema.
- Passos para reproduzir a vulnerabilidade.
- Versao do software afetada.
- Possivel impacto da vulnerabilidade.
- Qualquer proof-of-concept ou exploracao existente.

### O que esperar

Apos o reporte, a equipe do projeto:

1. Confirmara o recebimento em ate 48 horas uteis.
2. Fornecera uma avaliacao preliminar do reporte.
3. Trabalhara em uma correcao e se comunicara sobre o progresso.
4. Notificara quando a correcao estiver disponivel.

Em geral, trabalhamos para:

- Confirmar o problema e determinar as versoes afetadas.
- Auditar o codigo para encontrar problemas potencialmente similares.
- Preparar correcoes para todas as versoes ainda mantidas.
- Publicar uma divulgacao de seguranca apos a correcao estar disponivel.

## Politica de Divulgacao

Pedimos que nao divulgue publicamente a vulnerabilidade ate que:

- A correcao tenha sido publicada em uma release oficial.
- A equipe do projeto tenha tido tempo razoavel para resolver o problema,
  tipicamente 90 dias apos o reporte inicial.

## Escopo

Esta politica se aplica a todo o codigo-fonte do repositorio VFNews, incluindo
backend Java e frontend TypeScript/React.

## Consideracoes Finais

Obrigado por ajudar a manter o VFNews e seus usuarios seguros.
